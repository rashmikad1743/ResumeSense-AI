# ResumeSense AI

AI-powered resume reviewer built with Streamlit, Groq, and PDF parsing. Upload a PDF resume and receive constructive feedback generated by an LLM.

## Features
- **PDF parsing**: Extracts text from uploaded PDF resumes.
- **LLM feedback**: Uses Groq `llama-3.1-8b-instant` to generate a review.
- **Simple UI**: Streamlit app with progress indicators and error handling.

### File Descriptions
- **[app.py](app.py)**: Main Streamlit application. Handles file upload, UI layout, and orchestrates the review flow.
- **[groq_model.py](groq_model.py)**: Groq API client. Loads `GROQ_API_KEY` from environment/secrets and calls the LLM.
- **[pdf_reader.py](pdf_reader.py)**: PDF parsing module using `pypdf` to extract text from resumes.
- **[prompt.py](prompt.py)**: Prompt engineering. Builds the system and user prompts for the review.
- **[requirements.txt](requirements.txt)**: Lists all Python dependencies (streamlit, groq, pypdf, python-dotenv, google-generativeai).
- **[.env](.env)**: Local environment file for development (git-ignored). Add `GROQ_API_KEY=<your_key>` here.
- **[data/](data)**: Folder for sample or test resumes. Includes `sample.pdf` for quick testing.

## Configuration
This app looks for the `GROQ_API_KEY` in any of the following locations:
- **Environment variable** `GROQ_API_KEY`
- **.env** file loaded by `python-dotenv`
- **Streamlit secrets** `secrets.toml`

Options to set the key:

### Option A: Environment variable (current session)
```powershell
$Env:GROQ_API_KEY = "your_groq_api_key_here"
```

### Option B: Environment variable (persist for your user)
```powershell
[System.Environment]::SetEnvironmentVariable("GROQ_API_KEY","your_groq_api_key_here","User")
```

### Option C: .env file
Create a `.env` in the project root:
```
GROQ_API_KEY=your_groq_api_key_here
```

### Option D: Streamlit secrets
Create `.streamlit/secrets.toml`:
```
GROQ_API_KEY = "your_groq_api_key_here"
```

## Run the App
```powershell
streamlit run app.py
```

If the key is missing or invalid, the app shows a friendly error instead of crashing.

## How It Works
- The app reads the uploaded PDF via `pypdf` in [pdf_reader.py](pdf_reader.py).
- A review prompt is built in [prompt.py](prompt.py).
- The response is generated by Groq in [groq_model.py](groq_model.py) using model `llama-3.1-8b-instant`.
- Results are displayed in [app.py](app.py) under "Resume Review Report".

## Troubleshooting
- "Missing GROQ_API_KEY": Ensure the key is set via one of the configuration options above.
- Streamlit not starting: Verify the virtual environment is activated and dependencies are installed.
- PDF not parsing: Try a different file or confirm the PDF is text-based (not scanned).

## Notes
- Keep your API keys secure. Prefer `secrets.toml` for deployments.
- This repo is a minimal reference; adapt the prompt and output formatting to your needs.
